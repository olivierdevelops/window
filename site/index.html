Native desktop apps
with any backend

Write your UI in HTML/CSS/JS. Connect a backend in Python, Go, Rust, Node.js, or any language over a simple socket protocol. Or ship a WebAssembly module — zero runtime dependencies.

Quick Start

Get a window running in under a minute.

1. Build the binary

Requires Go 1.21+ and a C compiler (CGo). See Building from Source for platform deps.

git clone https://github.com/luowensheng/window
cd window
go build -o window .
# Optional: put it on your PATH
mv window /usr/local/bin/window

2. Scaffold a new project

window --init myapp
cd myapp

Creates window.yaml and static/index.html.

3. Run it

window window.yaml

Try a demo

window demos/hello/window.yaml        # static HTML
window demos/dashboard/window.yaml    # Chart.js via js_inject
window demos/file_explorer/window.yaml # native file system
window demos/terminal/window.yaml     # native OS exec
window demos/counter/window.yaml      # Python backend
💡 Set DEV_MODE=1 to start the HTTP server without opening a native window — useful for testing the backend from a browser.

Run Modes

Set mode: in window.yaml to choose how the window is opened.

modeDescription
server (default)Local HTTP server + optional backend subprocess. Best for most apps.
urlOpen a fixed URL directly. Useful for wrapping existing web apps.
proxyReverse-proxy to another dev server (e.g. npm run dev).
browserInject an HTML overlay into any web page via a closed shadow DOM.
controlledBackend subprocess creates, navigates, and destroys windows programmatically.
wasmA .wasm module is the backend — no subprocess, no socket.

Native Features

Expose file system, OS, dialog, and canvas APIs to the frontend — no backend process needed.

native_features:
  - fs       # NATIVE.fs.readFile / writeFile / readDir / watchFile
  - os       # NATIVE.os.exec / getEnv / platform / info
  - dialogs  # NATIVE.dialogs.openFile / saveFile / showMessage
  - canvas   # NATIVE.canvas.drawRect / drawText / clear

File System — NATIVE.fs

// Read a file as UTF-8 string
const text = await window.NATIVE.fs.readFile("/path/to/file.txt")

// Write a file (perm: Unix file mode, e.g. 420 = 0o644)
await window.NATIVE.fs.writeFile("/path/to/out.txt", "hello", 420)

// List directory → [{name, is_dir, size}]
const entries = await window.NATIVE.fs.readDir("/home/user/Documents")

// Watch a file for changes
window.NATIVE.fs.watchFile("/path/to/file", (newContent) => {
  console.log("changed:", newContent)
})

OS — NATIVE.os

// Execute a command
const r = await window.NATIVE.os.exec("git", ["log", "--oneline", "-5"])
console.log(r.stdout)   // stdout string
console.log(r.stderr)   // stderr string
console.log(r.error)    // error string (if non-zero exit)

// Environment
const home = await window.NATIVE.os.getEnv("HOME")

// Platform: "darwin" | "linux" | "windows"
const plat = await window.NATIVE.os.platform()

// { os: "darwin", arch: "arm64" }
const info = await window.NATIVE.os.info()

Dialogs — NATIVE.dialogs

// File picker → string[] of selected paths
const paths = await window.NATIVE.dialogs.openFile({
  title: "Open file",
  multi: false,
  filters: ["*.txt", "*.md"],
})

// Save dialog → string path
const dest = await window.NATIVE.dialogs.saveFile({ title: "Save as…" })

// Notification dialog
await window.NATIVE.dialogs.showMessage({
  title: "Done",
  message: "File saved.",
  kind: "info",  // "info" | "warn" | "error"
})

Backend Protocol

Any language that can connect to a Unix socket and read/write newline-delimited JSON can be a backend.

How it works

  1. Go creates a Unix socket and sets WINDOW_SOCK_PATH in the subprocess env.
  2. The subprocess connects to the socket.
  3. Both sides exchange newline-delimited JSON messages.

Message format

// Every message (both directions):
{
  "id":       "uuid",          // correlation ID
  "function": "handlerName",   // function / event name
  "data":     { ... },         // JSON payload
  "done":     true,            // true = final reply for this request
  "type":     "data",          // "data" | "eval"
  "code":     ""               // JS to eval when type == "eval"
}

Frontend JS API

// Call a backend function
BACKEND.call("add", { value: 5 }, ({ data, err }) => {
  if (err) { console.error(err); return; }
  console.log(data.value) // 6
})

// Subscribe to server-push events
BACKEND.onEvent("timer", ({ current_time }) => {
  document.getElementById("clock").textContent = current_time
})

Python client

Copy demos/counter/client.py into your project:

from client import App, ResponseWriter
import asyncio

app = App()

@app.handle("add")
async def add(req, rw):
    await rw.send({"value": req["value"] + 1})

# Push events to all frontends
async def tick():
    import time
    while True:
        await asyncio.sleep(1)
        await app.publish("timer", {"current_time": time.strftime("%H:%M:%S")})

async def main():
    await app.run()
    asyncio.create_task(tick())
    await asyncio.Event().wait()

asyncio.run(main())

Minimal Node.js backend

const net = require('net')
const client = net.connect(process.env.WINDOW_SOCK_PATH, () => {
  client.on('data', buf => {
    const msg = JSON.parse(buf)
    const reply = { id: msg.id, function: msg.function,
                    data: { result: 'ok' }, done: true }
    client.write(JSON.stringify(reply) + '\n')
  })
})

Server push

Send messages with an ID that starts with server: to push events to the frontend without a request:

await app.publish("notification", {"text": "Build complete!"})

◈
WASM Backend

Compile your backend to WebAssembly and ship it as a single binary — no subprocess, no socket, no runtime.

mode: wasm
wasm_backend: ./app.wasm
entry_path: ./static/index.html

Module contract

The .wasm module must export exactly two functions:

ExportSignatureDescription
alloc(size i32) → i32Allocate bytes; return pointer. Used by Go to write data before calling handle.
handle(fn_ptr, fn_len, data_ptr, data_len i32) → i64Process a request. Returns packed resp_ptr << 32 | resp_len.
init() → void (optional)Called once at module load. Use for global initialization.

TinyGo example

package main

import (
    "encoding/json"
    "unsafe"
)

var buf [1 << 20]byte
var heap [1 << 20]byte
var heapOff uintptr

//export alloc
func alloc(size int32) uintptr {
    ptr := uintptr(unsafe.Pointer(&heap[heapOff]))
    heapOff += uintptr(size)
    return ptr
}

//export handle
func handle(fnPtr, fnLen, dataPtr, dataLen int32) int64 {
    fn := string(unsafe.Slice((*byte)(unsafe.Pointer(uintptr(fnPtr))), fnLen))
    data := unsafe.Slice((*byte)(unsafe.Pointer(uintptr(dataPtr))), dataLen)
    var req map[string]any
    json.Unmarshal(data, &req)

    var resp map[string]any
    switch fn {
    case "add":
        v, _ := req["value"].(float64)
        resp = map[string]any{"value": v + 1}
    default:
        resp = map[string]any{"error": "unknown: " + fn}
    }

    b, _ := json.Marshal(resp)
    copy(buf[:], b)
    ptr := uintptr(unsafe.Pointer(&buf[0]))
    return int64(ptr)<<32 | int64(len(b))
}

func main() {}
tinygo build -o app.wasm -target wasi ./main.go
ℹ The WASM runtime is wazero — pure Go, no CGo, no external runtimes. WASI preview1 is instantiated automatically. Server-push events (BACKEND.onEvent) are not available in WASM mode — use polling instead.

Controlled Mode

Let a backend script create, navigate, eval JS in, and destroy windows programmatically.

mode: controlled
controlled_script: python3 controller.py

Command protocol

Go starts the script and sets WINDOW_CONTROL_SOCK_PATH. The script connects and sends commands:

// create_window
{"cmd":"create_window","id":"r1","params":{"title":"T","url":"https://x.com","width":800,"height":600}}
// → {"id":"r1","window_id":"win1"}

// navigate
{"cmd":"navigate","id":"r2","window_id":"win1","url":"https://y.com"}

// eval
{"cmd":"eval","id":"r3","window_id":"win1","js":"document.title='hi'"}

// close
{"cmd":"close","id":"r4","window_id":"win1"}

Python WindowManager

import asyncio, json, os, uuid

class WindowManager:
    def __init__(self):
        self.sock_path = os.getenv("WINDOW_CONTROL_SOCK_PATH")
        self.reader = self.writer = None
        self._pending = {}

    async def connect(self):
        for _ in range(20):
            try:
                self.reader, self.writer = await asyncio.open_unix_connection(self.sock_path)
                asyncio.create_task(self._recv_loop())
                return
            except (FileNotFoundError, ConnectionRefusedError):
                await asyncio.sleep(0.2)
        raise RuntimeError("could not connect to control socket")

    async def _recv_loop(self):
        async for line in self.reader:
            msg = json.loads(line)
            if (fut := self._pending.pop(msg.get("id"), None)):
                fut.set_result(msg)

    async def _send(self, cmd):
        cmd["id"] = uuid.uuid4().hex
        fut = asyncio.get_event_loop().create_future()
        self._pending[cmd["id"]] = fut
        self.writer.write((json.dumps(cmd) + "\n").encode())
        return await asyncio.wait_for(fut, 5)

    async def create_window(self, title, url, width=800, height=600):
        r = await self._send({"cmd":"create_window",
                               "params":{"title":title,"url":url,"width":width,"height":height}})
        return r["window_id"]

    async def navigate(self, win_id, url):
        await self._send({"cmd":"navigate","window_id":win_id,"url":url})

    async def eval(self, win_id, js):
        await self._send({"cmd":"eval","window_id":win_id,"js":js})

    async def close(self, win_id):
        await self._send({"cmd":"close","window_id":win_id})


async def main():
    wm = WindowManager()
    await wm.connect()
    win = await wm.create_window("Hello", "https://example.com", 700, 500)
    await asyncio.sleep(2)
    await wm.eval(win, "document.body.style.background='navy'")
    await asyncio.sleep(2)
    await wm.close(win)

asyncio.run(main())
⚠ Multiple simultaneous windows work best on Linux (GTK). macOS (Cocoa) has threading constraints that may cause unexpected behaviour with multiple webview instances.
