Scripts in .cs, compiled to JS

A .cs file is a tiny JavaScript-like program — let, fn, if/else, for…in, while, log. window app.cs compiles it to real JavaScript, drops it in a console-style window, and runs it. The whole language is defined as one Capy library — no parser hand-written, no toolchain.

how it works

Every keyword is one Capy function

Each statement form — fn, if, for — is a Capy block function. Blocks close with end; ${body} is re-indented and spliced into a JS template. if uses a multi-section block so the optional else branch only emits when it has content.

assets/capyscript.capy (the if/else rule)
function iff
    arg literal "if"
    arg capture cond tail
    block_sections else closer end     # `else` is an optional interior section
    write `if (${cond}) {
${indent 2 body}`
    if else                            # only emit `} else { … }` when non-empty
        write `} else {
${indent 2 else}}
`
    else
        write `}
`
    end
end
60-second tour

Recursion, loops, output

Keyword-led statements, blocks closed by end. log becomes console.log and is mirrored onto the window so output is visible with zero wiring.

fib.cs
const N = 15

fn fib(n)
    if n < 2
        return n
    else
        return fib(n - 1) + fib(n - 2)
    end
end

let i = 0
while i < N
    log "fib(" + i + ") = " + fib(i)
    do i = i + 1
end

Compiles to plain, readable JavaScript (real output):

GENERATED static/app.js const N = 15; function fib(n) { if (n < 2) { return n; } else { return fib(n - 1) + fib(n - 2); } } let i = 0; while (i < N) { console.log("fib(" + i + ") = " + fib(i)); i = i + 1; }

The one rule to remember: do

There is no catch-all line rule, so the grammar never guesses. Anything that isn't a keyword statement — an assignment, a method call, an arrow function — goes through do. That keeps every line unambiguous while letting full JavaScript through when you need it.

closures.cs — `do` passes raw JS through
fn makeCounter(start)
    do let n = start
    do return () => { n = n + 1; return n; }
end

const next = makeCounter(10)
log "first:  " + next()
log "second: " + next()

primes.cs
fn isPrime(n)
    let d = 2
    while d * d <= n
        if n % d == 0
            return false
        end
        do d = d + 1
    end
    return true
end
